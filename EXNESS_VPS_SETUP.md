# Exness VPS MT5 Deployment

Use this when running the DC Signals bridge on an Exness Windows VPS.

Production flow:

TradingView alert -> DC Signals Worker -> auto-trade queue -> MT5 EA on Exness VPS -> Exness MT5 order -> Worker ack.

The Worker is already configured in `paper` mode with MT5 polling. Keep it in paper until the VPS EA can poll and acknowledge commands.

## Files

- `mt5/DCSignalsExnessBridgeEA.mq5`: EA source.
- `DCSignalsExnessBridgeEA.ex5`: compiled EA generated by MetaEditor.
- `mt5/DCSignalsExnessBridgeEA.paper.example.set`: safe preset template without secret.

## Install On VPS

1. Log in to the Exness VPS with Remote Desktop.
2. Open MetaTrader 5 and sign in to the Exness trading account.
3. In MT5, open `File -> Open Data Folder`.
4. Put `DCSignalsExnessBridgeEA.ex5` into `MQL5/Experts/`.
5. Optional: put `DCSignalsExnessBridgeEA.mq5` into the same folder for future edits.
6. Put the `.set` preset into `MQL5/Presets/`.
7. Restart MT5 or right-click Navigator -> Refresh.

## Required MT5 Options

Open `Tools -> Options -> Expert Advisors`:

- Enable `Allow algorithmic trading`.
- Enable `Allow WebRequest for listed URL`.
- Add this URL:

```text
https://dc-signals-v91.cc559773.workers.dev
```

## Attach EA

1. Open any chart, preferably a symbol available in the Exness account.
2. Drag `DCSignalsExnessBridgeEA` onto the chart.
3. Load the paper preset.
4. Confirm:
   - `BridgeBaseUrl = https://dc-signals-v91.cc559773.workers.dev`
   - `BridgeSecret = same as admin Auto Trade Bridge Secret`
   - `LiveTrading = false`
   - `PollSeconds = 5`
   - `FallbackVolume = 0.01`
   - `AllowedStrategies = algo-pro-v1-4`
   - `SymbolSuffix = m` only if Exness symbols appear like `ETHUSDm` or `XAUUSDm`

The EA only executes commands whose `strategy_id` is included in `AllowedStrategies`. The Worker also has the same `auto_trade_allowed_strategies` whitelist, so non-approved TradingView strategies are blocked before they reach MT5.

## Paper Acceptance Test

After the EA is attached and WebRequest is allowed:

1. Keep Worker Auto Trade mode as `paper`.
2. Send one TradingView signal or create one manual signal.
3. Confirm admin Auto Trade shows command status moving from `queued`/`sent` to `acked`.
4. Confirm MT5 Journal shows `paper mode accepted`.
5. No real order should be opened while either Worker mode is `paper` or EA `LiveTrading` is `false`.

## Live Switch

Only after paper ack works:

1. In admin Auto Trade, set mode to `live`.
2. In MT5 EA inputs, set `LiveTrading = true`.
3. Keep `FallbackVolume = 0.01` and daily limit low for the first live test.
4. Send one small test signal.

Live execution requires both sides to be live:

- Worker Auto Trade mode: `live`
- EA input: `LiveTrading = true`

## Emergency Stop

Any one of these stops execution:

- Admin Auto Trade: disable auto trade.
- Worker Auto Trade mode: switch back to `paper`.
- MT5: turn off Algo Trading.
- MT5: remove the EA from the chart.
